# anti-patterns — the proofreader's memory of one writer's voice

The Corrector reads this **before** every run and updates it **after** each
human-audited run. Two lists, opposite jobs:

- **Recurring errors** — mistakes this writer actually makes. Give extra attention;
  flag on sight, with the fix.
- **Deliberate style** — intentional choices this writer makes that look like errors
  but aren't. **Never flag these.**

The point is precision on *one person's* prose: the proofreader gets quieter on
deliberate quirks and sharper on the real, repeated mistakes.

*(Seeded with common starting points — the every.to "anti-patterns blacklist" idea.
Replace these with what the human audit actually confirms. Each entry: the pattern →

---

## Recurring errors (flag + fix)

| Pattern | Fix |
| --- | --- |
| `its` / `it's` slips | possessive `its`; contraction `it's` (= "it is/has") |
| Hedge stacking ("I think maybe sort of") | cut to one qualifier or none |
| Fake symmetry ("on one hand… on the other") used to pad | delete the manufactured counterweight |
| Meandering throat-clearing intro | start at the first real sentence |
| Doubled small words across a line break ("the the", "to to") | delete the duplicate |

## Deliberate style (never flag)

| Choice | Why it's intentional |
| --- | --- |
| Em-dashes for asides | a load-bearing rhythm device, not a comma error |
| Sentence fragments for emphasis | deliberate cadence |
| Lowercase brand/term renderings (e.g. "tpot", "X") | the writer's intended orthography |
| Serial (Oxford) comma present | a consistent house style, not over-punctuation |
